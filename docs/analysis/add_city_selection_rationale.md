## 0. 프레임

### 0.1 공시적 vs 시계열 분리

이 프로젝트는 **두 트랙**을 명확히 분리한다.

**공시적 비교 (이 문서의 주제)**: 2026년 4월 동일 시점에서 4도시의 RDI 분포 비교. 리듬의 *차이가 어떻게 발생하고 어떻게 해석되는가*를 묻는다. Brenner/Urry/Sheller와 직접 접속.

**시계열 분석 (별도 문서)**: 창원 단독에서 1920년대 馬山日報 자동차 담론과 2026년 TAGO 관측의 교차. 100년의 리듬 재구성. Lefebvre *Éléments de rythmanalyse* + Elden 계보학과 접속.

두 축은 섞지 않는다. 서로 다른 저널·논증 구조·해석 장치를 요구한다.

### 0.2 공시적 비교의 4축 설계

4도시가 채워야 할 이론적 대조 축:

| 축 | 창원 | 성남 분당 | 세종 | 부산 영도 |
|---|---|---|---|---|
| **계획 시점** | 1970s 중화학 단지 | 1990s 1기 신도시 + 2010s 판교 | 2010s 행정중심 | 비계획(식민기~1950s 자생) |
| **수도권 여부** | 비수도권 광역 | **수도권 외곽 신도시** | 준수도권(분산 수도) | 비수도권 광역 |
| **자본 축적 논리** | 국가+중화학 | **민간 IT·금융 자본(판교)** | 국가+행정·교육 | 조선업→관광 전환 |
| **공간 형태** | 격자형 계획 | 계획 신도시+테크노벨리 | 방사+블록 계획 | 반도+봉래산 지형 |
| **인구 동학** | 정체 | 고밀도 유지+재개발 | 급성장 | 쇠락+부분 재생 |

**핵심 대조 구조**:

- **창원 vs 세종** — *국가 주도 계획도시의 세대 비교*. 1970s 중화학 vs 2010s 행정중심. 같은 국가 주도 계획이 40년 시차로 어떻게 다른 리듬을 생산하는가.
- **분당(판교) vs 창원·세종** — *민간 자본 계획 vs 국가 계획*. 같은 "계획" 범주 안에서 자본 논리의 차이가 리듬을 어떻게 다르게 생산하는가.
- **영도 vs 나머지** — *비계획(축적)의 리듬* vs *계획의 리듬*. 반도 지형이 계획 논리 외부에서 lived rhythm을 굴절.

**원안(강남)과의 차이**: 당초 계획한 "1970s 창원 vs 1970s 강남 영동지구"의 쌍(같은 시대·다른 자본 논리)은 TAGO 데이터 한계로 실행 불가. 성남 분당 채택으로 쌍이 "창원(국가 계획) vs 분당·판교(민간 자본 계획)"로 재구성되며, 시대 동기성은 포기하되 **자본 논리의 이항 대립**은 유지된다. 커미티 비판지리학자 위원이 지적한 "수도권 부재"는 분당의 수도권 외곽 신도시 성격으로 해소한다.

### 0.3 이론적 가설 (4도시 각각)

각 도시가 공시적 비교에서 제출하는 가설:

- **창원 (H-CW)**: 1970s 중화학 계획이 *자동차 전제 교외형 격자*를 생산 → BRT6000 dressage 독식 패턴 (Day 2 오전 관측으로 1차 확증)
- **성남 분당 (H-SN)**: 1990s 수도권 신도시+2010s 판교 테크노벨리가 *민간 자본 통근 리듬*을 생산 → 광역버스(서울 방면) variance 고(高), 내부 간선 dressage 중(中). 통근 피크에 vitality와 dressage가 시간대별로 전환
- **세종 (H-SJ)**: 2010s 신도시 계획이 *자동차 전제 방사형*을 최신 기술로 재시행 → 창원 BRT6000보다 더 순응하는 dressage 패턴 예상. 창원과의 세대 차이 검증
- **부산 영도 (H-YD)**: 반도 지형+다리 병목이 prescribed-lived 격차를 *구조적으로* 생산 → magnitude 고(高) + variance 고(高). vitality_query 집중 발현 예상

**4도시 교차 발견 1 — regional_fix 축의 4변주 (Day 2 오후 확정)**: 네 도시의 regional_fix 노선이 각자 독립적 이론 위치를 점유함이 Code의 영도 정찰에서 확정되었다. 창원 710(**결핍** — 광역 교통 부재의 증상적 대체) / 세종 1004(**필수성** — 대전·청주 연계가 생활의 조건) / 분당 9407(**특권** — 서울 방면 고밀 통근 회랑) / 영도 113(**왜곡** — 광역이 존재하되 지형에 굴절). 이 4변주는 Brenner의 rescaling 이론이 한국 도시에서 자기 변형을 찾는 네 유형을 경험적으로 드러내며, Day 3~4 이중 언어 이론 브리프의 핵심 분석 자산으로 활용한다.

**4도시 교차 발견 2 — automobility_capture 노선의 정류소 스케일 차이**: 창원 BRT6000(77 stn, 기존 도시에 덧붙여진 BRT) / 세종 B2(15 stn, 계획 단계부터 압축 통합된 BRT) / 영도 8번(89 stn, 다리 병목이 있는 간선). 정류소 수 자체가 도시 공간 구조의 흔적을 드러내며, ARDI 계산 이전에 이미 automobility 배치의 질적 차이를 암시한다. 성남 분당 101번은 해당 축 확인 필요.

## 1. 성남시 분당구 (cityCode 31020)

### 1.0 선정 경위

당초 이 프레임의 수도권 위치에는 **서울 강남구**가 배치되었으나(1970s 영동지구와 창원의 동시대 대조), TAGO API가 서울시 데이터를 커버하지 않는다는 사실이 Day 2 정오에 확인되었다. 서울 시내버스 정보는 TOPIS/서울 열린데이터광장의 별도 API 체계로 운영되며, 해커톤 5일 스코프에서 새 ingest 모듈을 추가하는 것은 타임라인을 붕괴시킨다. 따라서 TAGO 내부 대체로 **성남 분당구**를 채택한다. 강남의 이론적 역할 일부(수도권 자본 집중, 금융·IT 결절)는 판교 테크노벨리가 부분 대체하며, 시대 동기성("1970s 쌍")은 포기한다.

### 1.1 공간 논리

성남시는 1960년대 서울 도심 재개발 과정에서 **철거민 집단이주지**(현 수정구·중원구)로 출발한 도시다. 이 초기 성남은 국가가 서울의 과밀과 빈곤을 공간적으로 외주화한 장소였다. 그 위에 1989년부터 **분당 1기 신도시**(노태우 정부 200만호 건설 계획)가 얹혔고, 2003년에는 **판교 테크노벨리**가 착공되어 2009년 개원했다. 현재 성남시는 한 행정경계 안에 **1960s 이주지·1990s 신도시·2010s 테크노벨리**의 세 시대 층위가 응축된 공간이다.

분당구에 한정하면 공간 논리는 다음과 같다:

- **업무 축**: 판교 테크노벨리 (카카오·네이버·NC소프트 등 IT 대기업 본사·연구소)
- **주거 축**: 분당 1기 신도시 (서현·정자·수내·야탑·미금), 판교 신도시(2010s)
- **교육 축**: 분당 학원가 (서현·정자), 전국 상위권 학군으로 자리매김
- **교통 축**: 분당선·신분당선·경강선·GTX-A·8호선(예정) 철도 다층 + 경부고속도로·분당수서도시고속화도로
- **통근 축**: 서울 강남·사당 방면 광역버스 고밀도 (판교·분당 → 서울 도심의 특권적 통근 회랑)

이 구성은 **민간 자본 주도 계획도시의 리듬적 특성**을 보인다. 창원이 국가 중화학 분업에 맞춘 *생산자 리듬*이라면, 분당·판교는 *지식노동자+통근 자본*의 리듬이다. RDI 관점에서 **분당 내부 이동은 상대적으로 정시성 유지(dressage), 서울 방면 광역 통근은 정체 민감성 고(高)(vitality)**의 이중 구조가 예상된다.

### 1.2 이론적 배치 — 3노선 축

창원과 동일 원칙(`route_selection_rationale.md` 참조): **automobility 포획 / 광역 부재의 증상적 대체 / 생활 리듬 잔여** 삼각. 분당에서는 다음과 같이 재번역된다.

#### (a) automobility 포획 대표 — 분당 내부 간선

판교역·정자역·서현역·야탑역 등 분당 내부 주요 결절을 종단하거나 환승 연결하는 간선버스. 신도시 격자형 도로에서 비교적 정시 운영 — **dressage 축 대표**(창원 BRT6000 대응).

**후보 노선(조회 필요)**:
- 성남 시내버스 **101번** (판교·분당 관통 간선)
- 성남 시내버스 **220번** (서현·야탑 내부)
- 분당 간선 **330번** 또는 유사 번호

#### (b) 광역 통근 회랑 — 서울 방면 직행좌석·광역급행

분당·판교에서 서울 강남·사당·잠실 방면으로 집중되는 **광역버스**. prescribed 배차는 상대적으로 타이트하지만(5~15분, 피크 때), lived는 경부고속도로·강남대로 정체에 구조적으로 노출. 통근 피크에 magnitude·variance 모두 급등 예상 — **창원 710(광역 부재의 증상적 대체)과 이론적으로 대응하되 방향이 반대**. 창원은 광역 교통 부재의 결핍, 분당은 광역 자본 결절의 과잉.

**후보 노선(조회 필요)**:
- **9407번** (분당~강남역 직행좌석)
- **9408번** (오리~서울역 광역)
- **3330번** (판교~강남역 직행좌석)
- **4000번** (수원~서울역) — 분당 경유 확인 필요

#### (c) 마을버스 / 지선 — 생활 리듬 잔여

분당 내부 주거단지·학원가·지하철역을 연결하는 **마을버스**. 대치동만큼은 아니지만 분당 서현·정자·수내 학원가의 오후·저녁 리듬을 담는 노선. 창원 271(저밀도 희박)과 달리 **고밀도 주거·교육 리듬의 생활권 대표** — vitality 축이되 혼잡형.

**후보 노선(조회 필요)**:
- 성남 마을버스 **33번** (서현·이매 주거단지)
- 성남 마을버스 **누리2번** (정자·수내 학원가)
- 성남 마을버스 **8번** 또는 **판교마을 1번**

### 1.3 Code 핸드오프 (Cowork → Code)

다음 3노선 후보에 대해 TAGO `BusRouteInfoInqireService`로 routeid·routeno·startnodenm 등 메타 조회 요청:

```yaml
city: seongnam_bundang
city_code: 31020
candidates:
  - axis: automobility_capture
    query: "성남 간선 101 또는 220 또는 330 (분당 내부 관통)"
    expect_role: "dressage 대표 — 분당 신도시 내부 간선, BRT6000 대응"
  - axis: regional_fix
    query: "광역급행 9407, 9408, 3330 중 (판교·분당 → 서울 강남·도심)"
    expect_role: "특권적 광역 통근 회랑 — 창원 710의 반대 극, 과잉형"
  - axis: residue
    query: "성남 마을버스 33, 누리2, 판교마을 1 (분당 내부 주거·학원)"
    expect_role: "고밀도 생활 리듬 — 창원 271의 반대 극(희박→혼잡)"
```

Code는 **실제 TAGO에서 조회 가능한 노선 중** 위 원칙에 가장 가까운 3개를 선정. 분당구 내부를 실제로 관통하는지 `route_stations.nodeid`로 분당구 OSM 경계 교차 확인. 조회 불가·정보 부족 시 Cowork에 피드백.

## 2. 세종특별자치시 (cityCode 12)

### 2.1 공간 논리

세종시는 **2010년대 국가 주도 계획도시의 최신 버전**이다. 2003년 노무현 정부의 신행정수도 계획이 2004년 헌법재판소 위헌 결정으로 좌절된 뒤, 2005년 행정중심복합도시 특별법으로 우회 경로가 열렸다. 2007년 착공, 2012년 특별자치시 출범, 이후 중앙부처의 순차 이전이 진행 중이다. 행정·정치적 의도는 수도권 과밀 분산이었으나, 실제 도시 설계는 **자동차 전제 방사형 신도시**의 체계적 구현이다.

분석 대상은 **행정중심복합도시(행복도시)**를 구성하는 1~6 생활권이다. 기존 농촌 지역(조치원·연기)과 구분되는 계획 구역. 공간 구성:

- **핵심축**: 정부세종청사 → 호수공원 → 종합병원 중심부
- **6개 생활권 방사 배치**: 1생활권(고운동) / 2생활권(아름·종촌) / 3생활권(도담·어진) / 4생활권(한솔·새롬) / 5생활권(대평·보람) / 6생활권(해밀·한별)
- **BRT 중심 체계**: 한국 최초로 도시 설계 단계부터 BRT를 통합. B1(오송역~반석역, 대전 유성 연계), B2(반석~세종청사~오송역) 순환
- **녹지 50%** 계획 + 도로 위계는 여전히 자동차 중심
- **연령 구조**: 공무원·교사 중심 젊은 세대, 평균 연령 한국 최저 수준
- **통근 패턴**: 세종↔대전·청주 생활권 연계 (광역버스 고밀도)

창원(1970s 중화학)과 비교하면 세종은 **같은 국가 주도 계획이되 40년 후의 계획 언어**로 쓰였다. 창원이 산업노동력 재배치를 위한 *생산자 도시*였다면, 세종은 *행정·지식노동 도시*다. RDI 관점에서 핵심 가설은 **세종의 BRT가 창원 BRT6000보다 더 순응적 dressage 패턴**을 보일 것이라는 점이다. 40년의 계획 기술 누적이 *prescribed에 대한 lived의 충실도*를 어떻게 높였는가 — 이것이 두 도시 비교의 이론적 초점이다.

**대중교통 분류 체계에 새겨진 계획 논리 (Day 2 오후 발견)**: 세종 3노선 중 automobility_capture(B2)와 regional_fix(1004) 모두 TAGO `routetp="광역버스"`로 분류된다. 창원이 *BRT(예외) + 좌석(광역 기능) + 일반(시내)*의 이질적 혼성이라면, 세종은 *광역 통합을 기본 형질로 전제*하고 구시가(조치원) 일반버스만을 예외로 남긴다. 이 분류 체계의 차이는 단순 행정 편의가 아니라 **두 도시의 계획 철학이 교통 체계 편제까지 새겨놓은 흔적**으로 읽을 수 있다. Day 3~4 이중 언어 브리프에서 활용 예정.

### 2.2 이론적 배치 — 3노선 축

#### (a) automobility 포획 대표 — 세종 BRT

세종 BRT는 전용차로·정시성·도시계획 통합 운영이라는 점에서 **한국 automobility의 계획적 최전선**이다. 창원 BRT6000이 기존 도시에 *덧붙여진* BRT라면, 세종 BRT는 *도시 설계의 일부로 태어난* BRT. dressage가 가장 깊이 구조화된 축.

**후보 노선**:
- **B1번** (오송역 ~ 반석역, 대전 유성 연계) — 간선 중 간선
- **B2번** (반석역 ~ 정부세종청사 ~ 오송역) — 순환
- **B0번** 또는 유사 급행 — 존재 확인 필요

#### (b) 광역 통근 회랑 — 세종↔대전·청주

세종은 행정도시이지만 완결된 생활권을 아직 구축하지 못했다. 의료·쇼핑·교육 상당 부분을 **대전 유성·서구** 또는 **청주 오송·흥덕구**에 의존한다. 이 광역 연계 리듬이 실제 세종 거주자의 일상을 지배. 분당의 "서울 방면 통근 회랑"과 구조적으로 대응되지만 방향이 다중이다 — **창원 710(광역 부재의 증상적 대체)과 이론적 쌍**. 창원은 광역 교통 부재의 결핍, 세종은 광역 기능 연계의 필수성.

**후보 노선**:
- **201번** (세종 ↔ 대전 반석역) — 세종-대전 간선
- **202번** 또는 유사 (세종청사 ~ 대전 유성) 
- **801번** (세종 ↔ 청주 오송역) — 세종-청주 간선
- **990번** 또는 광역급행 — 존재 확인 필요

#### (c) 생활권 내부 지선 — 희박형 잔여

세종 행복도시는 1~6 생활권이 단계적으로 개발되어, **개발 초기 생활권(2·3생활권)**과 **최근 입주 생활권(6생활권)** 사이의 인프라 격차가 존재한다. 특히 6생활권 해밀·한별 방면은 배차가 드물고 대중교통 의존도가 낮음. 창원 271(외곽 희박형)과 이론적 대응 — 다만 창원 271이 *구시가 외곽의 느슨함*이라면, 세종의 지선은 *신도시 주변부의 느슨함*. 둘 다 계획 체계의 *잔여*이되 성격이 상반된다.

**후보 노선**:
- **1004번** 또는 유사 (생활권 순환 지선)
- **1002번** (신도시 외곽 연결)
- **조치원 방면 지선** (구시가↔신도시 연결, 예: 551번)

## 3. 부산광역시 영도구 (cityCode 21, 영도 필터링)

### 3.1 공간 논리

영도는 한국 근대화 과정에서 **계획 외부에 남아 축적된 도시**의 전형이다. 1876년 부산 개항과 함께 외국인 거류지·해운 근거지로 개방되었고, 식민기 일본인 어업·조선업 중심지로 성장했으며, 해방 후 1950년대 한국전쟁 피란민 정착지이자 한국 조선업의 발상지(대한조선공사 등)가 되었다. 1970년대 이후 국가 산업정책은 울산·거제로 조선업 주축을 옮겼고, 영도는 중소 수리조선·선박부품 산업의 잔여 회로로 남았다. 2010년대에 이르러 **흰여울문화마을·절영해안산책로·태종대**를 축으로 하는 관광·문화 젠트리피케이션이 도시 조직을 재편 중이다.

공시적 비교의 다른 세 도시(창원·분당·세종)가 모두 *계획 논리의 산물*이라면, 영도는 *계획의 외부에서 축적된 도시 조직*이다. 이 위상 차이는 RDI 분석에 결정적이다.

**영도의 세 지형-인프라 제약**:

1. **반도 지형 + 봉래산**: 영도는 엄밀히 섬이 아니라 반도-섬 복합체다. 북서쪽에 영도대교(1934)·부산대교(1980) 2개 다리로만 도심 연결. 내부는 봉래산(395m)이 중앙을 차지, 해안선을 따라 도로망 형성. 격자형 계획이 불가능한 지형.
2. **다리 병목**: 두 다리의 통과 용량이 영도 전체 육상 교통의 병목. 러시아워·관광 성수기에 대기시간이 버스 배차의 prescribed를 구조적으로 교란.
3. **해풍·조수**: 해안선 도로(절영로 등)가 조수·해풍에 노출. 태풍기 운행 제한 빈번.

RDI 관점의 핵심 가설 (H-YD): **영도에서는 prescribed와 lived의 격차가 지형-인프라 제약에 의해 *구조적으로* 생산된다.** 창원·분당·세종처럼 도시 내부의 *계획 논리* 차이가 RDI를 결정하는 것이 아니라, 도시 외부 조건(지형·다리·기상)이 RDI에 개입. 이는 Sheller(2020) *Island Futures*가 말한 *섬 모빌리티의 관계적 구성*과 정확히 조응한다 — 섬은 폐쇄된 단위가 아니라 다리·배·기상이 접합된 열린 연결망이다.

**분석 범위 한정**: TAGO cityCode 21은 부산광역시 전체 풀이다. 영도 한정 분석을 위해서는 `route_stations.nodeid`에 기록된 좌표가 **영도구 OSM 행정경계 polygon 내부**에 포함되는 노선만 필터링한다. Code가 osmium extract로 `data/processed/osm/busan_yeongdo.pbf`를 사전 추출해둔 상태(Day 2 오후).

### 3.2 이론적 배치 — 3노선 축

영도의 3노선 선정 원칙은 창원·분당·세종과 동일한 삼각(automobility 포획 / 광역 부재의 증상적 대체 / 생활 잔여)이지만, 영도의 지형 제약 때문에 **"광역"이 곧 "다리 건너"**를 의미한다. 영도 외부로 나가려면 반드시 다리를 건너야 하고, 그 병목이 광역 연결의 prescribed-lived 격차를 결정한다.

#### (a) automobility 포획 대표 — 다리 관통 간선

영도 내부에서 영도대교 또는 부산대교를 건너 부산 도심(중구·서구)으로 연결되는 간선버스. 영도의 대부분 주요 간선이 이 유형에 속하며, 다리 통과 여부가 운행의 구조적 변수. 창원 BRT6000과 대응하되 **"다리가 있는 BRT"** 성격 — 이론적으로 **prescribed 타이트 + lived 병목 민감**의 이중 구조.

**후보 노선 (실조회 필요)**:
- **8번** (영도 태종대 ↔ 서구 범일동, 영도대교 경유)
- **11번** (영도 동삼동 ↔ 중구 남포동)
- **66번** (영도 청학동 ↔ 부산 도심)
- **85번** 또는 **88번** 등 영도 관통 간선

#### (b) 광역 부재의 증상적 대체 — 원거리 광역버스

영도 내부 정류장에서 부산 외곽(해운대·기장·북구·강서구) 또는 김해·양산 방면으로 장거리 운행하는 노선. 다리 병목 + 부산 내부 이동 거리 + 외곽 연결의 삼중 부하로 **variance 고(高), magnitude 불안정** 예상. 창원 710(광역 없음의 결핍)과도, 세종 1004(광역 필수성)와도 다른 *제3의 극* — **광역이 존재하되 지형에 의해 왜곡된 극**.

**후보 노선 (실조회 필요)**:
- **186번** (영도 ↔ 부산 북구 또는 강서구, 장거리)
- **113번** (영도 관통 후 부산 외곽 방면)
- **190번** 또는 유사 장거리 노선

#### (c) 생활 잔여 — 영도 내부 마을버스

영도 내부만 운행하며 봉래산 주변·해안선·달동네를 연결하는 **마을버스**. 봉래산 경사·좁은 골목·관광지 진입로 등 물리적 조건이 배차에 직접 영향. 창원 271(시가지→외곽)·분당 누리2(고밀 생활권)·세종 551(구시가 지선)의 **"지형 잔여"** 변형 — 계획 체계 바깥이 아니라 *지형 자체가 체계를 허용하지 않은 공간*의 잔여.

**후보 노선 (실조회 필요)**:
- **영도 마을 1번** (봉래산 남측)
- **영도 마을 2번** (청학·동삼)
- **영도 마을 3번** (절영로·흰여울문화마을)

## 5. 경주시 (Day 4 add-city 시연)

### 5.1 위치 — 공시적 비교 바깥의 시연 사례

경주는 공시적 비교 4도시(창원·분당·세종·영도)에 속하지 않는다. **Day 4 `rhythmscape add-city --name gyeongju` 라이브 데모 사례**로 별도 위상을 가진다. 이 섹션은 "왜 경주인가"와 "어떻게 시연하는가"에 한정된다.

### 5.2 선정 근거

사천·진주를 원안에서 시연 사례로 유지했으나, Day 2 오후 공시적 비교 재설계로 두 도시의 시연 정당성이 상실되었다(원안이 "경남권 4도시 유형학"에 종속되어 있었기 때문). 시연 도시의 역할을 *글로벌 리치 + 도구 일반성 증명*으로 재규정하면서, 다음 세 조건으로 경주를 단독 선정했다.

1. **외국인 심사위원 글로벌 인지도 ★★★★★**: UNESCO 세계유산(불국사·석굴암·경주역사지구), K-heritage 상징. 비한국어권 리뷰어가 30초 안에 도시 맥락을 이해.
2. **공시적 4도시와 축 중복 없음**: 고대 도시 조직 + 관광 중심 + 소도시 automobility는 창원(1970s 계획)·분당(1990s 민간 자본)·세종(2010s 행정)·영도(비계획 반도)와 완전히 독립된 위상.
3. **기술적 일반성 시연**: 해커톤 심사의 Demo(25%)와 Depth(20%)에서 "이 도구는 UNESCO 세계유산 도시에서도 작동한다"는 서사를 30초 라이브로 전달 가능.

### 5.3 공간 논리

경주는 신라 천년 수도(57 BCE~935 CE)의 도시 조직이 현재 도시계획과 중첩되는 공간이다. 시내(황남·황오·성건)에 대릉원·첨성대·계림 등 고대 유산이 산재하고, 외곽에 불국사·석굴암이 있으며, 보문관광단지가 관광 리듬의 핵심 결절이다. 공간 구성의 특이성:

- **문화재 보호 규제**: 시내 곳곳에 고도제한, 건축 규제, 굴착 제한 등 *계획의 자유를 제한하는 층*이 있음. 다른 4도시가 계획 논리가 *작동한* 공간이라면, 경주는 계획이 *제약된* 공간.
- **관광 리듬의 이중 층위**: 평일/주말, 성수기/비성수기, 국내/국제 관광의 중첩. prescribed는 정적이되 lived는 관광 계절성에 직접 지배됨.
- **소도시 + 광역 면적**: 인구 약 25만, 면적 1,324km² (전국 시 단위 4위). 시내버스가 보문~시내~불국사~감포를 장거리 연결.
- **동해안 축**: 감포·양남 방면 해안선. 영도와 다른 의미의 "해양-내륙 접합" 구조.

이 도시에서 RDI·ARDI를 돌린다면 가설은 명확하다. **ARDI는 도시 중심부에서는 낮고(문화재 보호 + 보행 중심 관광), 외곽 도로에서는 극단적으로 높을 것**(관광지 연결 도로의 자동차 지배). PRM은 문화재 구역에서 *예외적 보행 우세* 패턴을 보일 가능성 높음. 다만 Day 4 시연은 **ARDI 단독**으로 한정하고, 이 가설은 해커톤 이후 확장 과제로 남긴다.

### 5.4 Day 4 라이브 데모 서사

**실행 명령**:
```
rhythmscape add-city --name gyeongju --mode minimal
```

**자동 실행 단계 (목표 30~60초)**:
1. `config/cities.yaml`에 경주 섹션 자동 생성 (템플릿 상속)
2. OSM `south-korea-latest.osm.pbf`에서 경주시 경계로 서브셋 추출 (osmium extract)
3. TAGO cityCode 37130 경주시에서 route 메타 조회 (시내버스 전체, 약 50-80개 예상)
4. ARDI 5항(road_space_ratio, walkable_surface_ratio, speed_regime, car_throughput proxy, transit_frequency) 계산
5. ARDI 격자 HTML 히트맵 생성 → `reports/gyeongju/ardi_map.html`

**해설 내러티브 (데모 비디오 음성)**:
> "같은 도구가 UNESCO 세계유산 도시에서 작동합니다. 경주의 ARDI 지도는 시내 문화재 구역에서는 낮은 automobility 지배를, 외곽 관광지 연결 도로에서는 극단적 지배를 드러냅니다. *지표는 답이 아니라 질문입니다* — 역사도시에서 자동차 리듬은 어떻게 문화재 보호와 타협하는가? 이 질문을 즉시 생성할 수 있다는 것, 이것이 Rhythmscape가 4도시 심층 분석을 넘어 *도구로 작동하는* 이유입니다."

### 5.5 Code 핸드오프 (Day 4 사전 준비)

Day 2 오후 현재 경주 관련 작업은 **대기 상태**. Day 4 데모 직전 다음 작업 일괄 실행:

```yaml
city: gyeongju
city_code: 37130
mode: minimal          # ARDI 단독
realtime_tago: false   # 실시간 수집 불필요 — route 메타만

setup_sequence:
  - osmium_extract: "south-korea-latest.osm.pbf → gyeongju.pbf"
  - tago_meta_only: "resolve_routes --city gyeongju --dump (route + route_stations + stations anchor만)"
  - ardi_compute: "ARDI 5항 격자 계산, 500m grid"
  - html_render: "reports/gyeongju/ardi_map.html (이중 언어)"

theoretical_framing_notes: |
  - 경주는 공시적 비교 4도시에 속하지 않는다. 시연 전용.
  - ARDI 해설에서 "문화재 보호가 계획의 자유를 제약한 공간"이라는 위상 차이를 명시.
  - RDI·PRM은 돌리지 않는다 (해커톤 이후 확장 과제).
```

**API 호출 예산**: 경주 route 메타 조회는 1회성 (수십 호출 이내). 실시간 수집 없으므로 TAGO 일일 한도에 영향 없음.

**데모 리허설**: Day 4 오전 리허설에서 실제 시간 측정 후 스토리보드 조정.
